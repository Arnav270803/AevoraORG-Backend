# Aevora Backend

Express + TypeScript backend for Aevora authentication, projects, ad drafts, asset records, and pipeline job tracking.

## Stack

- Runtime: Node.js
- Server: Express
- Language: TypeScript, compiled to JavaScript in `dist`
- Database ORM: Prisma
- Database: PostgreSQL
- Google identity verification: `google-auth-library`
- JWTs: `jose`
- Request validation: `zod`

## Implemented APIs

Protected routes require:

```http
Authorization: Bearer AEVORA_ACCESS_JWT
```

### Health and auth

### `GET /api/health`

Returns a basic service health response.

### `POST /api/auth/google`

Completes Google Sign-In.

Request:

```json
{
  "credential": "GOOGLE_ID_TOKEN_FROM_FRONTEND"
}
```

Response:

```json
{
  "user": {
    "id": "local-user-id",
    "email": "user@example.com",
    "name": "User Name",
    "avatarUrl": "https://..."
  },
  "accessToken": "AEVORA_ACCESS_JWT",
  "refreshToken": "AEVORA_REFRESH_JWT"
}
```

### `POST /api/auth/refresh`

Rotates a refresh token and returns a new access token plus refresh token.

Request:

```json
{
  "refreshToken": "AEVORA_REFRESH_JWT"
}
```

### `POST /api/auth/logout`

Revokes the submitted refresh token.

Request:

```json
{
  "refreshToken": "AEVORA_REFRESH_JWT"
}
```

### `GET /api/auth/me`

Returns the authenticated user.

### Projects

### `POST /api/projects`

Creates a user-owned project.

Request:

```json
{
  "name": "Glow Serum Launch",
  "description": "Short-form ad tests for paid social",
  "metadata": {
    "workspace": "spring-campaign"
  }
}
```

### `GET /api/projects`

Lists projects owned by the authenticated user, including ad counts.

### `GET /api/projects/:projectId`

Returns one owned project with its ads and per-ad counts.

### Ads

### `POST /api/projects/:projectId/ads`

Creates a draft ad under an owned project.

Request:

```json
{
  "title": "Glow Serum TikTok Draft",
  "productName": "Glow serum bottle",
  "brandName": "Aevora Beauty",
  "category": "Skincare",
  "referenceNotes": "Front label visible, clean bathroom counter, soft morning light.",
  "platform": "tiktok",
  "aspectRatio": "9:16",
  "durationSeconds": 15,
  "creativeBrief": {
    "hook": "Hydrated skin in one step"
  }
}
```

`title` is optional. If omitted, the backend uses `productName` or `Untitled ad`.

### `GET /api/ads/:adId`

Returns one owned ad with its project summary, assets, pipeline jobs, step runs, and render outputs.

### `PATCH /api/ads/:adId`

Updates an owned ad draft. Accepts the create fields plus optional `status`.

Allowed statuses:

- `DRAFT`
- `READY`
- `GENERATING`
- `COMPLETED`
- `FAILED`
- `ARCHIVED`

### Assets

### `POST /api/ads/:adId/assets`

Registers an asset record for an owned ad. This does not upload binary data yet; it prepares a provider-agnostic storage record through the local placeholder storage provider.

Request:

```json
{
  "kind": "PRODUCT_IMAGE",
  "fileName": "front-bottle.png",
  "mimeType": "image/png",
  "sizeBytes": 1240000,
  "metadata": {
    "source": "create-ad-ui"
  }
}
```

Allowed kinds:

- `PRODUCT_IMAGE`
- `REFERENCE_IMAGE`
- `LOGO`
- `BRAND_GUIDE`
- `OTHER`

### `GET /api/ads/:adId/assets`

Lists asset records for an owned ad.

### Pipeline Jobs

### `POST /api/ads/:adId/pipeline-jobs`

Creates a queued pipeline job record for an owned ad. This backend does not execute AI/video generation.

Request:

```json
{
  "type": "AD_GENERATION",
  "priority": 0,
  "provider": "future-pipeline",
  "requestPayload": {
    "workflow": "default-product-ad"
  },
  "steps": [
    {
      "name": "shot_generation",
      "sequence": 1
    },
    {
      "name": "script_generation",
      "sequence": 2
    }
  ]
}
```

Allowed job types:

- `AD_GENERATION`
- `RENDER_EXPORT`

Initial job status is `QUEUED`.

### `GET /api/ads/:adId/pipeline-jobs`

Lists pipeline jobs for an owned ad, including step runs and render outputs.

### `GET /api/pipeline-jobs/:jobId`

Returns one owned pipeline job, including ad summary, step runs, and render outputs.

## Data Model Summary

- `User`: local account created from Google Sign-In.
- `Project`: user-owned container for ad work.
- `Ad`: draft or generated ad concept under a project.
- `Asset`: metadata and provider-agnostic storage pointer for product/reference assets.
- `PipelineJob`: queued/running/completed job record for a future external pipeline service.
- `PipelineStepRun`: ordered per-step status tracking under a pipeline job.
- `RenderOutput`: provider-agnostic output pointer for videos, images, scripts, or metadata generated by a job.

Ownership is enforced through `Project.ownerId`; all ad, asset, pipeline-job, step, and render-output lookups are scoped back to the authenticated user's projects.

## Storage Boundary

Business modules do not depend on AWS, S3, or any storage vendor. Asset creation uses the `StorageProvider` interface in `src/infrastructure/storage`.

Current implementation:

- `LocalPlaceholderStorageProvider`
- Creates a deterministic local-style storage key and URL.
- Does not write files or serve binary assets.

Future implementations can replace this provider with S3, Supabase Storage, Cloudflare R2, local disk, or another service without changing project/ad business logic.

## Future Pipeline Service Contract

The future pipeline service should treat this backend as the source of truth for:

- ad draft inputs and asset metadata
- pipeline job IDs and requested job payloads
- step status updates
- render output records

In the next pipeline loop, add protected internal/service APIs for the pipeline worker to:

- claim or read queued `PipelineJob` records
- update `PipelineJob.status`, `startedAt`, `completedAt`, `resultPayload`, and error fields
- create/update `PipelineStepRun` records as stages start and finish
- create `RenderOutput` records for generated videos, images, scripts, and metadata

Keep provider-specific details in JSON payloads or infrastructure adapters, not in project/ad business modules.

## Required Setup Values

Fill these in `.env` before running locally:

- `DATABASE_URL`: PostgreSQL connection string.
- `GOOGLE_CLIENT_ID`: Google Identity Services web client ID.
- `JWT_ACCESS_SECRET`: strong random secret, at least 32 characters.
- `JWT_REFRESH_SECRET`: different strong random secret, at least 32 characters.
- `FRONTEND_ORIGIN`: local frontend origin, likely `http://localhost:5173`.
- `BACKEND_URL`: local backend URL, likely `http://localhost:4000`.
- `LOCAL_STORAGE_PUBLIC_BASE_URL`: optional local placeholder asset URL base. Defaults to `${BACKEND_URL}/local-assets` when omitted.

Then run:

```bash
npm install
npm run prisma:generate
npm run prisma:migrate
npm run dev
```

## Completed Foundation

- Google OAuth/JWT auth is preserved.
- User-owned projects can be created and listed.
- Ad drafts can be created, read, and updated under owned projects.
- Asset records can be registered against owned ads through a provider-agnostic storage interface.
- Pipeline job and step-run records can be created/listed/read for owned ads.
- Render output table is ready for future generated assets.
- Zod validates request bodies and route params.
- Prisma migration `20260627075113_backend_foundation` adds the backend foundation tables and enums.

## Local Docker Database

The local database runs Postgres in Docker and matches the default `DATABASE_URL` in `.env.example`:

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/aevora
```

Start the database:

```bash
npm run db:up
```

Apply Prisma migrations:

```bash
npm run prisma:migrate
```

Useful database commands:

```bash
npm run db:logs
npm run db:down
npm run db:reset
```

`db:reset` removes the Docker volume and deletes local database data.
